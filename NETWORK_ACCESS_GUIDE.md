# 同一 WiFi 下无法访问的解决方案

## 🔍 问题原因

Vite 开发服务器默认只监听 `localhost`（127.0.0.1），不允许外部设备访问。

## ✅ 已完成的配置

### 1. 后端 Flask 配置 ✅
已经配置为监听 `0.0.0.0`（允许所有网络接口访问）：
```python
app.run(host='0.0.0.0', port=5000, debug=True)
```

### 2. 前端 Vite 配置 ✅
已修改 `vite.config.ts` 允许外网访问：
```typescript
server: {
  host: '0.0.0.0', // 允许外网访问
  port: 5173,
}
```

### 方法一：使用 start.sh 脚本（推荐）


## 🔧 排查步骤

如果仍然无法访问，请按以下步骤排查：

### 1. 检查服务是否正常启动

```bash
# 检查后端进程
ps aux | grep "python app.py"

# 检查前端进程
ps aux | grep "npm run dev"

# 检查端口监听
netstat -tlnp | grep -E "5000|5173"
```

**正常输出示例：**
```
tcp        0      0 0.0.0.0:5000           0.0.0.0:*               LISTEN
tcp        0      0 0.0.0.0:5173           0.0.0.0:*               LISTEN
```

### 2. 检查防火墙状态

```bash
# 查看防火墙状态
sudo ufw status

# 如果防火墙启用，需要开放端口
sudo ufw allow 5000/tcp
sudo ufw allow 5173/tcp
```

### 3. 测试本地访问

在服务器上测试：
```bash
# 测试后端
curl http://localhost:5000/api/state

# 测试前端
curl http://localhost:5173
```

### 4. 检查 WiFi 网络设置

确保其他设备：
- ✅ 连接到同一个 WiFi
- ✅ 在同一网段（10.148.x.x）
- ✅ 没有被路由器隔离（AP 隔离）

### 5. 从其他设备测试

在另一台设备上执行：
```bash
# ping 测试
ping 10.148.0.105

# 端口连通性测试（如果安装了 telnet）
telnet 10.148.0.105 5173
```

## ⚠️ 常见问题

### Q1: 提示 "连接超时"
**原因**：服务未正确监听 0.0.0.0 或防火墙阻止

**解决**：
1. 确认 vite.config.ts 已配置 `host: '0.0.0.0'`
2. 重启前端服务
3. 检查防火墙设置

### Q2: 提示 "无法访问此网站"
**原因**：端口被阻止或服务未启动

**解决**：
1. 检查服务是否正常运行
2. 确认端口 5173 和 5000 正在监听
3. 尝试关闭防火墙临时测试

### Q3: 只能本机访问，其他设备不行
**原因**：路由器启用了 AP 隔离

**解决**：
1. 登录路由器管理页面
2. 关闭 "AP 隔离" 或 "客户端隔离" 功能
3. 重启路由器

### Q4: Vite 启动后显示错误
**可能错误**：端口被占用

**解决**：
```bash
# 查找占用端口的进程
lsof -i :5173

# 杀死占用进程
kill -9 <PID>

# 或者修改 vite.config.ts 使用其他端口
server: {
  host: '0.0.0.0',
  port: 5174, // 改为其他端口
}
```

## 🔐 安全提示

⚠️ **注意**：允许外网访问开发服务器可能存在安全风险：

1. **仅限内网使用**：不要在公网开放这些端口
2. **开发环境专用**：不要在生产环境使用 `debug=True`
3. **及时关闭**：不使用时及时停止服务
4. **密码保护**：如需长期开放，建议添加认证
